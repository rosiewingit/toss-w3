# Toss UI/UX 가이드 체크리스트

이 문서는 Apps-in-Toss 플랫폼에서 미니앱을 출시하기 전 반드시 확인해야 하는 UI/UX 가이드라인 체크리스트입니다. **모든 항목을 통과해야 배포가 가능합니다.**

## 📋 체크리스트 사용 방법

1. 각 섹션의 항목을 순서대로 확인하세요
2. 통과한 항목은 `[x]`로 체크하세요
3. 실패한 항목은 `[ ]`로 남겨두고 수정 후 다시 확인하세요
4. **모든 필수 항목(★)을 통과해야 배포 가능합니다**

---

## 1. 필수 설정 및 패키지 (필수 ★)

### 1.1 TDS (Toss Design System) 설치 및 사용
- [ ] `@toss/tds-mobile` 패키지가 설치되어 있는가? (`@apps-in-toss/web-framework >= 1.0.0`인 경우)
- [ ] `@toss-design-system/mobile` 패키지가 설치되어 있는가? (`@apps-in-toss/web-framework < 1.0.0`인 경우)
- [ ] TDS 컴포넌트를 실제로 사용하고 있는가? (커스텀 컴포넌트만 사용하면 안 됨)
- [ ] 샌드박스 앱을 통해 TDS가 정상 동작하는지 테스트했는가? (로컬 브라우저에서는 테스트 불가)

### 1.2 WebView 설정
- [ ] `granite.config.ts`에서 `webViewProps.type`이 올바르게 설정되어 있는가?
  - 비게임: `"partner"`
  - 게임: `"game"`
- [ ] `bridgeColorMode` 설정이 제거되었는가? (2024년 12월 18일부터 deprecated)

### 1.3 프로젝트 설정
- [ ] `granite.config.ts`의 `appName`이 콘솔에 등록한 앱 이름과 일치하는가?
- [ ] `brand.displayName`이 한글로 올바르게 설정되어 있는가?
- [ ] `brand.primaryColor`가 RGB HEX 형식(`#3182F6`)으로 설정되어 있는가?
- [ ] `brand.icon`이 올바른 이미지 URL로 설정되어 있는가? (또는 빈 문자열 `''`)

## 2. UX 라이팅 가이드 (필수 ★)

### 2.1 해요체 사용
- [ ] 모든 문구가 해요체로 작성되어 있는가? (상황, 맥락 불문하고)
- [ ] "~입니다", "~합니다" 같은 높임말이 없는가?
- [ ] "~시겠어요?", "~시나요?" 같은 과도한 경어가 없는가?

### 2.2 능동형 문장 사용
- [ ] "됐어요" → "했어요"로 수정되었는가?
- [ ] "~었" 형태가 적절히 제거되었는가? (예: "완료되었어요" → "완료했어요")
- [ ] 수동형 문장이 특정 상황(서비스 종료, 기간 만료, 사용자 안심 등)에서만 사용되는가?

### 2.3 긍정적 표현 사용
- [ ] "안 돼요", "없어요" 같은 부정적 표현이 최소화되었는가?
- [ ] "~하면 할 수 있어요" 같은 긍정적 표현으로 대체되었는가?
- [ ] 에러 메시지가 긍정적이고 해결 방법을 제시하는가?

### 2.4 캐주얼한 경어 사용
- [ ] "~시"가 제거되었는가? (예: "확인하시겠어요?" → "확인할까요?")
- [ ] "계시다" → "있다"로 변경되었는가?
- [ ] "여쭈다" → "확인하다", "묻다"로 변경되었는가?
- [ ] "께" → "에게"로 변경되었는가?

### 2.5 명사+명사 형태 피하기
- [ ] 한자어 명사를 동사 형태로 풀어썼는가? (예: "결제 진행" → "결제를 진행")
- [ ] "{명사}가 {명사}해서" 형태로 풀어썼는가?

### 2.6 예외 규칙 준수
- [ ] "되어요" → "돼요"로 통일했는가? (모바일 화면 공간 고려)
- [ ] 특정 상황에서만 경어를 사용했는가? (사용자 맥락 활용, 상황 추정, 선의 필요 시)
- [ ] 부정형 문장이 필요한 경우에만 사용했는가? (서비스 정책상 사용 불가, 일부 기능만 사용 불가, 사용자 안심)

## 3. 디자인 가이드라인 (필수 ★)

### 3.1 피그마 가이드 준수
- [ ] 모든 화면의 최상단에 `Mobile_Navigation` 컴포넌트를 사용했는가?
- [ ] 또는 `Screen` 컴포넌트를 사용했는가? (필수 컴포넌트 포함)
- [ ] 텍스트 입력 시 피그마 기본 텍스트 대신 `Mobile_Text` 컴포넌트를 사용했는가?
- [ ] 오른쪽 패널에서만 속성을 조작했는가? (캔버스에서 직접 수정하지 않았는가?)
- [ ] 화면을 가로 375px 기준으로 작업했는가?
- [ ] 화면 상단에 `Mobile_Top`, 그 아래에 `Mobile_ListRow`를 사용했는가?

### 3.2 TDS 컴포넌트 사용
- [ ] 🔵 파란색 원 표시 컴포넌트(개발 문서에 코드 제공)를 우선 사용했는가?
- [ ] 🟣 보라색 원 표시 컴포넌트 사용 시 별도 구현 계획이 있는가?
- [ ] 🔴 빨간색 원 표시 컴포넌트(커스텀 불가)를 수정하지 않았는가?
- [ ] 🌈 무지개 표시 컴포넌트 사용 시 별도 코드 작성 계획이 있는가?

### 3.3 컴포넌트 간격 및 레이아웃
- [ ] TDS 컴포넌트의 기본 패딩을 활용했는가?
- [ ] 간격 조절이 필요할 때 오토레이아웃의 gap 또는 `Padding・Margin` 컴포넌트를 사용했는가?
- [ ] 컴포넌트를 분리해서 사용할 때 신중하게 판단했는가? (개발 시 별도 코드 필요)

## 4. 접근성 (Accessibility) (필수 ★)

### 4.1 색상 대비
- [ ] 일반 텍스트(18pt 미만 또는 14pt 볼드)의 색상 대비가 4.5:1 이상인가? (WCAG 2.0 AA)
- [ ] 큰 텍스트(18pt 이상 또는 14pt 볼드)의 색상 대비가 3:1 이상인가? (WCAG 2.0 AA)
- [ ] 배경과 텍스트 색상의 대비를 확인했는가?

### 4.2 가시성 및 포커스
- [ ] 활성화 가능한 모든 요소가 포커스 가능한가?
- [ ] `opacity:0`, `z-index` 조정이 콘텐츠 숨기기용으로만 사용되는가?
- [ ] 콘텐츠 숨기기 시 `hidden`, `visibility`, `display` 스타일 속성을 사용했는가?

### 4.3 텍스트 동등성
- [ ] 텍스트 이미지 사용을 자제했는가?
- [ ] 모든 UI 컴포넌트에 `alt`, `aria-label` 등 텍스트가 제공되는가?
- [ ] Form 컨트롤에 `<label>`이 포함되어 있는가?
- [ ] 스크린리더 사용자를 고려했는가?

### 4.4 기타 접근성 요구사항
- [ ] 콘텐츠가 가로/세로 단일 방향으로 제한되지 않는가?
- [ ] 제목 요소들이 계층구조를 유지하는가? (`<h1>`, `<h2>`, `<h3>` 등)
- [ ] 키보드만으로 모든 기능을 사용할 수 있는가?

## 5. 모바일 최적화 (필수 ★)

### 5.1 뷰포트 설정
- [ ] `index.html`에 올바른 viewport meta 태그가 설정되어 있는가?
  ```html
  <meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no, viewport-fit=cover" />
  ```
- [ ] `user-scalable=no`가 설정되어 있는가? (앱인토스 환경)
- [ ] `viewport-fit=cover`가 설정되어 있는가? (노치 대응)

### 5.2 터치 최적화
- [ ] 터치 영역이 최소 44x44px인가? (Apple HIG 권장)
- [ ] 버튼과 클릭 가능한 요소 간 충분한 간격이 있는가?
- [ ] `-webkit-tap-highlight-color: transparent`가 설정되어 있는가?
- [ ] `touch-action`이 적절히 설정되어 있는가?

### 5.3 반응형 디자인
- [ ] 다양한 화면 크기에서 정상 동작하는가?
- [ ] 가로 모드와 세로 모드를 모두 지원하는가? (필요한 경우)
- [ ] 작은 화면에서도 콘텐츠가 잘 보이는가?

### 5.4 성능 최적화
- [ ] 이미지가 최적화되어 있는가? (WebP, 적절한 크기)
- [ ] 불필요한 리렌더링이 없는가?
- [ ] 애니메이션이 부드럽게 동작하는가? (60fps)
- [ ] 초기 로딩 시간이 적절한가?
